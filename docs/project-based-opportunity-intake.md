# TechSubbies Project-Based Opportunity Intake

## What changed

The intake workflow now treats a customer request as a project with multiple resource stages rather than one generic job.

A project can contain separate requirements for:

- survey
- first fix cabling
- installation support
- rack build
- UC validation
- audio commissioning
- video commissioning
- LED/video wall commissioning
- project management
- IT support or IT migration work

## Multi-stage engineer reuse

Engineers can appear in more than one stage of the same project.

When a candidate appears across multiple non-overlapping stage searches, the UI can label them:

- ALREADY ON-SITE OPTION

This helps customers see when a multi-skilled engineer could cover several stages and helps engineers win more work from complete profiles.

When the stage dates overlap, the UI labels the issue:

- SCHEDULE CLASH

## Still separate from skills

This workflow only matches:

- role fit
- must-have skills
- nice-to-have skills
- product/platform experience

It does not decide:

- insurance
- documents
- tools/testers
- final availability
- day rate
- contract terms
