# TechSubbies IT Skills and Product Knowledge Model

## Separation of concerns

The IT engineer profile should separate:

1. Role skills - what the engineer can do.
2. Product awareness and experience - which vendors, platforms and technologies they know.
3. Setup profile - tools, testers, documents, insurance, site cards, equipment and vehicle ownership.
4. Availability profile - location, working radius, calendar, weekends and holidays.

## Product knowledge recommendation

Product knowledge should be managed as a separate tagging layer, not duplicated inside every IT role profile.

Each product or technology tag should support these experience levels:

- Aware
- Assisted
- Installed
- Configured
- Administered
- Commissioned
- Programmed
- Troubleshot
- Certified

## Why

A Service Desk Technician, Cloud Administrator and Security Analyst may all know Microsoft 365, but their experience is different.

Examples:

- Service Desk Technician: Microsoft 365 - Troubleshot
- Microsoft 365 Administrator: Microsoft 365 - Administered
- Security Analyst: Microsoft Defender - Troubleshot / Administered
- Consultant: Azure - Aware / Designed / Specified

This creates better matching than a simple yes/no vendor checkbox.

## Role profile structure

Each IT role contains:

- Role summary
- Suitable work types
- Must-have skill summary
- Nice-to-have skill summary
- Expected knowledge
- Role aspects
- Typical evidence
- Product knowledge tag suggestions
- Skill groups split into must-have and nice-to-have

## Boundary

Keep tools, testers, hardware ownership, vehicles, site cards, insurance and official documentation in the setup/profile section, not in skills.
