# TechSubbies Compliance and Certification Model

## Core rule

Basic access stays open. Certificates increase trust and match confidence. Certificates only become mandatory when the project, site, customer, country or role genuinely requires them.

## Why this matters

TechSubbies should support lower-earning engineers and new freelancers, while still giving companies the ability to filter for safety, quality, insurance, industry competence and project-specific requirements.

Avoid one flat certificate list. Use role-specific and region-specific evidence.

## Compliance categories

1. Safety / site access
2. AV industry credentials
3. IT / networking / cybersecurity credentials
4. Manufacturer / platform training
5. Project management credentials
6. Insurance documents
7. Company quality standards
8. Regional compliance
9. Background checks
10. Work evidence

## Requirement levels

| Level | Use |
|---|---|
| Mandatory for this project | Required because the project, site, customer, country or work type needs it |
| Strongly preferred | Improves suitability and confidence significantly |
| Useful evidence | Helpful, but should not block the match |
| Company-level only | Applies mainly to companies or resourcing firms |
| Region-specific | Applies only in certain countries or customer environments |
| Not relevant | Should not be requested by default |

## Examples

### Basic AV labour

Do not require CTS, CCNA, PRINCE2 or ISO by default.

Useful or project-specific evidence:

- Site safety card if the site requires it
- Public liability insurance
- Manual handling
- Working at height
- Project photos or references

### AV Systems Engineer / Commissioner

Relevant evidence:

- AVIXA CTS / CTS-I
- Public liability insurance
- Professional indemnity insurance
- Dante certification
- Manufacturer training
- Commissioning evidence

### AV-over-IP Engineer

Relevant evidence:

- CompTIA Network+
- Cisco CCNA
- Dante certification
- SDVoE training
- Manufacturer AVoIP training
- Network project references

### UC / Meeting Room Engineer

Relevant evidence:

- Microsoft 365 / Teams administration
- AVIXA CTS
- Q-SYS / Biamp / Dante
- Meeting room handover examples
- Background check where project-specific

### IT Support Engineer

Relevant evidence:

- CompTIA A+
- Microsoft 365
- Security awareness
- Project references

### Network Engineer

Relevant evidence:

- Network+
- CCNA
- CCNP
- Security+
- Professional indemnity insurance

### Project Manager

Relevant evidence:

- PRINCE2
- PMP
- Delivery references
- RAMS awareness
- Company quality systems where relevant

### Resourcing Company

Paid resourcing companies need stronger verification:

- Business verification
- Public liability insurance
- Employers' liability insurance where applicable
- Professional indemnity insurance
- ISO 9001 / ISO 45001 / ISO 27001 where relevant
- Cyber Essentials where relevant
- Clear resourcing-company disclosure

## Trust badges

Trust badges should be awarded from verified documents, not just profile claims.

Examples:

- ID verified
- Country verified
- Insurance verified
- Safety verified
- Certified
- Manufacturer certified
- Company quality verified
- Project management verified
- Background checked
- Platform verified
- Work-proven

## Data rules

Store only what is needed.

Recommended stored fields:

- Credential key
- Verification status
- Verified country/region where relevant
- Issuer
- Issue date
- Expiry date
- Last checked date
- Verification provider result
- Public/private visibility setting

Avoid storing full identity documents or sensitive background-check details longer than necessary.

## Product rule

The compliance model must improve match quality without making TechSubbies inaccessible.

Mandatory should mean genuinely mandatory.
